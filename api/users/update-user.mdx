## Update User

<api-endpoint>
  <api-endpoint-request>
    <api-endpoint-request-method>PUT</api-endpoint-request-method>
    <api-endpoint-request-url>/api/users</api-endpoint-request-url>
  </api-endpoint-request>
</api-endpoint>

Update an existing user account.

<Note>
This endpoint requires `ROLE_ADMIN` permissions.
</Note>

### Request Body

<ParamField body="id" type="integer" required>
  User ID to update
</ParamField>

<ParamField body="login" type="string" required>
  Username (must match existing or be unique)
</ParamField>

<ParamField body="email" type="string" required>
  Email address (must match existing or be unique)
</ParamField>

<ParamField body="firstName" type="string" required>
  User's first name
</ParamField>

<ParamField body="lastName" type="string" required>
  User's last name
</ParamField>

<ParamField body="authorities" type="array" required>
  Array of role names
</ParamField>

<ParamField body="activated" type="boolean" required>
  Whether the account is activated
</ParamField>

<ParamField body="langKey" type="string">
  Language preference
</ParamField>

<ParamField body="imageUrl" type="string">
  Profile image URL
</ParamField>

### Response

<ResponseField name="id" type="integer">
  User ID
</ResponseField>

<ResponseField name="login" type="string">
  Username
</ResponseField>

<ResponseField name="email" type="string">
  Email address
</ResponseField>

<ResponseField name="firstName" type="string">
  First name
</ResponseField>

<ResponseField name="lastName" type="string">
  Last name
</ResponseField>

<ResponseField name="activated" type="boolean">
  Activation status
</ResponseField>

<ResponseField name="authorities" type="array">
  Assigned roles
</ResponseField>

<RequestExample>
```bash cURL

  -H "Authorization: Bearer eyJhbGciOiJIUzUxMiJ9..." \

    "lastName": "Doe",
```


## Validation Rules

### Email Uniqueness
Email addresses must be unique across all users. The check is case-insensitive.

### Login Uniqueness
Usernames must be unique. The check is case-sensitive.

### Admin Protection
The system prevents removing `ROLE_ADMIN` from the last remaining admin user.

### Password Requirements
- Minimum length: 4 characters
- Maximum length: 100 characters

## Update Examples

### Add Analyst Role

```python Python
import requests

def add_analyst_role(api_url, token, user_id):
    # Get current user
    headers = {"Authorization": f"Bearer {token}"}
    response = requests.get(f"{api_url}/users/{user_id}", headers=headers)
    user = response.json()
    
    # Add ROLE_ANALYST if not present
    if "ROLE_ANALYST" not in user["authorities"]:
        user["authorities"].append("ROLE_ANALYST")
        
        # Update user
        headers["Content-Type"] = "application/json"
        response = requests.put(f"{api_url}/users", headers=headers, json=user)
        
        if response.status_code == 200:
            print(f"Added ROLE_ANALYST to user {user['login']}")
        else:
            print(f"Failed to update user: {response.text}")
```

### Deactivate User

```python Python
import requests

def deactivate_user(api_url, token, user_id):
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    }
    
    # Get user
    response = requests.get(f"{api_url}/users/{user_id}", headers=headers)
    user = response.json()
    
    # Set activated to false
    user["activated"] = False
    
    # Update
    response = requests.put(f"{api_url}/users", headers=headers, json=user)
    return response.status_code == 200
```

## Best Practices

1. **Verify Before Update**: Get current user data before updating
2. **Preserve Roles**: Don't accidentally remove important roles
3. **Admin Protection**: Always maintain at least one admin user
4. **Email Validation**: Ensure email addresses are valid before updating
5. **Audit Changes**: Log all user modifications for compliance
